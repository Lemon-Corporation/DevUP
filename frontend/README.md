# DevUp: Приложение для программистов

DevUp - приложение для программистов, помогающее подготовиться к техническим собеседованиям в игровой форме.

## Вклад разных программистов

Проект разрабатывался командой из 4 программистов, каждый из которых внёс свой вклад в последовательном порядке:

### Программист 1: Базовая структура приложения

**Выполненные задачи:**
1. Создание основного скелета приложения, включая главный файл main.dart
2. Разработка экрана загрузки (Splash Screen)
3. Настройка основной темы и стилей приложения
4. Создание навигации и основных экранов
5. Настройка файлов конфигурации для iOS и Android
6. Разработка базовых виджетов: DarkBackground, Navigation
7. Реализация Values и Constants для переиспользования в приложении

### Программист 2: Улучшение пользовательского интерфейса

**Выполненные задачи:**
1. Разработка стилизованных форм ввода и кнопок
2. Создание экранов авторизации и регистрации
3. Реализация кастомных форм и поисковых компонентов
4. Добавление анимированных переходов между экранами
5. Создание компонентов Onboarding для новых пользователей
6. Разработка виджетов в Shapes (app_settings_icon, settings_strip)
7. Улучшение отображения на разных устройствах

### Программист 3: Логика и данные

**Выполненные задачи:**
1. Разработка моделей данных для курсов, заданий, пользователей
2. Создание контроллеров для управления состоянием приложения
3. Реализация экранов профиля пользователя
4. Разработка экранов дашборда и поиска
5. Создание сервисов для работы с данными
6. Настройка взаимодействия между компонентами через GetX
7. Реализация логики аутентификации

### Программист 4: Геймификация и финальные улучшения

**Выполненные задачи:**
1. Разработка системы игрового прогресса и наград
2. Создание компонентов для отображения достижений
3. Реализация ежедневных задач и уведомлений
4. Разработка экранов для обучающих материалов
5. Создание компонентов для отображения прогресса пользователя
6. Реализация интервью-симуляций для подготовки к собеседованиям
7. Оптимизация производительности приложения

## Структура проекта

Каждая папка в проекте содержит часть кода, разработанную соответствующим программистом:

- `programmer-1/lib` - код, разработанный первым программистом
- `programmer-2/lib` - код, добавленный вторым программистом
- `programmer-3/lib` - код, добавленный третьим программистом 
- `programmer-4/lib` - код, добавленный четвертым программистом

Итоговый проект представляет собой комбинацию работы всех четырех программистов.

## Структура проекта

В проекте используется следующая структура:

- **lib/** - основной код приложения
  - **Screens/** - экраны приложения
  - **widgets/** - переиспользуемые виджеты
  - **Controllers/** - GetX контроллеры
  - **Data/** - модели данных и сервисы
  - **Values/** - константы, стили, цвета
  - **Utils/** - утилиты и хелперы

5. Введите "Flutter: Select Device" и выберите устройство

## Архитектура проекта

При разработке использовался модифицированный паттерн MVC+S, который обеспечивает:

- Разделение ответственности между компонентами
- Тестируемость кода
- Удобство поддержки
- Масштабируемость

### Структура данных

#### Модели

1. **UserModel**
    - User - информация о пользователе, достижения, прогресс
    - UserCourseProgress - отслеживание прогресса по курсам
    - Achievement - модель достижений пользователя
    - DailyChallenge - ежедневные задания

2. **CourseModel**
    - Course - информация о курсе (название, описание, модули)
    - Module - модули курса с уроками
    - Lesson - отдельные уроки внутри модулей
    - Instructor - информация о преподавателях

3. **TaskModel**
    - Task (абстрактный) - базовый класс для всех заданий
    - TestTask - тестовые задания с вариантами ответов
    - CodeAnalysisTask - задания на анализ кода
    - AlgorithmTask - алгоритмические задачи
    - Question - вопросы в тестовых заданиях
    - Example - примеры для алгоритмических задач

#### Сервисы

1. **DataService**
    - Центральное хранилище данных (GetX)

2. **AppServices**
    - Инициализирует сервисы + внедрение зависимостей

#### Контроллеры

1. **AppStateController**
    - Управляет состоянием приложения
    - Предоставляет реактивные переменные состояния
    - Единая точка доступа к состоянию приложения

#### Утилиты

**. **AppExtensions**

- Вспомогательные функции для UI
